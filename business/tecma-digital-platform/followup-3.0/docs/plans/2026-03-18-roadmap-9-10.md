# Followup 3.0 — Roadmap 7/10 → 9/10

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Portare Followup 3.0 da "startup ben attrezzata" a CRM enterprise real estate production-hardened, con CI/CD reale, osservabilità, coverage robusta e feature core completate.

**Architecture:** Modular monolith fullstack (Express BE + React FE + MongoDB). Il piano segue 5 fasi sequenziali con gate di uscita misurabili. Le prime 3 fasi sono puramente tecniche (hardening); le ultime 2 aggiungono feature enterprise. Non mescolare mai hardening e nuove feature nella stessa sprint.

**Tech Stack:** Node.js 18 + Express + TypeScript + MongoDB (BE) | React 18 + Vite + Tailwind + TypeScript (FE) | Vitest + Supertest + Playwright (test) | Docker + GitHub Actions (CI/CD) | Socket.io (realtime) | Pino (logging)

**KPI target (definizione concreta di "9/10"):**
- BE coverage ≥ 70% sui moduli core (auth, clients, apartments, requests, calendar, workflow)
- CI success rate ≥ 95% per 30 giorni consecutivi
- Change failure rate < 15%
- MTTR < 60 minuti
- P95 API core < 400ms (misurato da k6)
- E2E flussi critici pass rate ≥ 95%
- Nessun file "hub" > 600 righe nei path critici (v1.ts 1095L, App.tsx 612L, followupApi.ts 928L)
- Zero secret hardcoded, policy env enforced

**Regola critica (NON negoziabile):** Per le prime 3 fasi, nessuna nuova feature va in produzione. Feature freeze. Chi viola questa regola fa slittare il piano.

**Step 2: Creare `.github/workflows/ci.yml`**

La pipeline deve eseguire in ordine:
1. Lint (BE + FE)
2. Build (BE + FE)
3. Test unit + API (BE)
4. Test unit + component (FE)
5. Coverage check con soglie minime (fail se BE < 40%, FE < 65%)
6. (Solo su `main`) E2E Playwright smoke

```yaml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  be:
    name: Backend CI
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: be-followup-v3
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: be-followup-v3/package-lock.json
      - run: npm ci
      - run: npm run test:lint
      - run: npm run build
      - run: npm run test:coverage
        env:
          NODE_ENV: test
      - name: Coverage gate (BE ≥ 40%)
        run: |
          STMT=$(cat coverage/coverage-summary.json | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); const j=JSON.parse(d); console.log(j.total.statements.pct)")
          echo "BE coverage: $STMT%"
          node -e "if (parseFloat('$STMT') < 40) { console.error('BE coverage below 40%'); process.exit(1); }"

  fe:
    name: Frontend CI
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: fe-followup-v3
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: fe-followup-v3/package-lock.json
      - run: npm ci
      - run: npm run build
      - run: npm run test:coverage
      - name: Coverage gate (FE ≥ 65%)
        run: |
          STMT=$(cat coverage/coverage-summary.json | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); const j=JSON.parse(d); console.log(j.total.statements.pct)")
          echo "FE coverage: $STMT%"
          node -e "if (parseFloat('$STMT') < 65) { console.error('FE coverage below 65%'); process.exit(1); }"
```

### Task 2.2: Correlation ID per request tracing

> Senza correlation ID non puoi tracciare una request attraverso i log. 3 righe di codice, impatto enorme.

**Files:**
- Create: `be-followup-v3/src/routes/correlationId.middleware.ts`
- Modify: `be-followup-v3/src/routes/v1.ts`

**Step 1: Creare il middleware**

```typescript
// src/routes/correlationId.middleware.ts
import { Request, Response, NextFunction } from 'express';
import { randomUUID } from 'crypto';

export const correlationIdMiddleware = (req: Request, res: Response, next: NextFunction) => {
  const id = (req.headers['x-correlation-id'] as string) || randomUUID();
  req.headers['x-correlation-id'] = id;
  res.setHeader('x-correlation-id', id);
  next();
};
```

**Step 2: Applicarlo prima di tutto in v1.ts**

```typescript
import { correlationIdMiddleware } from './correlationId.middleware.js';
// Prima riga del router, prima di qualsiasi altro middleware:
router.use(correlationIdMiddleware);
```

**Step 3: Passarlo al logger per includerlo in ogni log**

Nel Pino HTTP middleware in server.ts, aggiungere il campo `correlationId` nel genReqId:

```typescript
app.use(pinoHttp({
  logger,
  genReqId: (req) => req.headers['x-correlation-id'] as string,
}));
```

**Step 4: Scrivere il test**

```typescript
// be-followup-v3/src/routes/correlationId.middleware.test.ts
import { describe, it, expect } from 'vitest';
import request from 'supertest';
import express from 'express';
import { correlationIdMiddleware } from './correlationId.middleware.js';

describe('correlationIdMiddleware', () => {
  const app = express();
  app.use(correlationIdMiddleware);
  app.get('/test', (req, res) => res.json({ id: req.headers['x-correlation-id'] }));

  it('genera un UUID se non presente nell header', async () => {
    const res = await request(app).get('/test');
    expect(res.headers['x-correlation-id']).toMatch(/^[0-9a-f-]{36}$/);
  });

  it('propaga il correlation ID ricevuto', async () => {
    const res = await request(app).get('/test').set('x-correlation-id', 'test-id-123');
    expect(res.headers['x-correlation-id']).toBe('test-id-123');
  });
});
```

```typescript
// src/components/ErrorBoundary.tsx
import { Component, ErrorInfo, ReactNode } from 'react';

interface State { hasError: boolean; error?: Error; }

export class ErrorBoundary extends Component<{ children: ReactNode; fallback?: ReactNode }, State> {
  state: State = { hasError: false };

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, info: ErrorInfo) {
    // TODO Fase 4: inviare a error tracking (Sentry/simile)
    console.error('ErrorBoundary caught:', error, info);
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback ?? (
        <div className="flex flex-col items-center justify-center h-screen gap-4">
          <h1 className="text-xl font-semibold">Qualcosa è andato storto</h1>
          <p className="text-muted-foreground text-sm">{this.state.error?.message}</p>
          <button onClick={() => this.setState({ hasError: false })} className="btn-primary">
            Riprova
          </button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

```typescript
import { Server as HttpServer } from 'http';
import { Server as SocketServer, Socket } from 'socket.io';
import { verifyAccessToken } from '../core/auth/token.service.js';
import { logger } from './logger.js';

let io: SocketServer;

export const initSocketIO = (httpServer: HttpServer) => {
  io = new SocketServer(httpServer, {
    cors: { origin: process.env.APP_PUBLIC_URL, credentials: true },
    path: '/v1/ws',
  });

  io.use((socket: Socket, next) => {
    const token = socket.handshake.auth.token;
    try {
      const payload = verifyAccessToken(token);
      socket.data.userId = payload.userId;
      socket.data.workspaceId = payload.workspaceId;
      next();
    } catch {
      next(new Error('Unauthorized'));
    }
  });

  io.on('connection', (socket: Socket) => {
    const room = `workspace:${socket.data.workspaceId}`;
    socket.join(room);
    logger.info({ userId: socket.data.userId, room }, 'Socket connected');
    socket.on('disconnect', () => logger.info({ userId: socket.data.userId }, 'Socket disconnected'));
  });

  return io;
};

export const getIO = () => {
  if (!io) throw new Error('Socket.io not initialized');
  return io;
};
```

```typescript
// magic-link.service.ts
export const generateMagicLink = async (clientId: string, workspaceId: string): Promise<string> => {
  const token = randomUUID();
  const expiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000); // 7 giorni
  await db.collection('tz_magic_links').insertOne({ token, clientId, workspaceId, expiresAt, used: false });
  return `${process.env.APP_PUBLIC_URL}/portal?token=${token}`;
};

export const exchangeMagicLink = async (token: string) => {
  const record = await db.collection('tz_magic_links').findOne({ token, used: false });
  if (!record || new Date(record.expiresAt) < new Date()) throw new HttpError(401, 'Link scaduto o non valido');
  await db.collection('tz_magic_links').updateOne({ token }, { $set: { used: true } });
  // Restituisce JWT customer-scope (read-only, solo i dati del cliente)
  return generateCustomerToken(record.clientId, record.workspaceId);
};
```
