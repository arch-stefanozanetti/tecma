# fe-tecma-design-system
 
## Overview
The fe-tecma-design-system project is a design system library built for Tecma Solutions. This library supports component development, styling, and deployment processes across different environments.
 
## Branching Strategy
 
### Commit Messages
This project uses `commitlint` to enforce commit message standards. Ensure your commit messages follow the conventional commit format:
 
- **feat**: A new feature
- **fix**: A bug fix
- **docs**: Documentation changes
- **style**: Code style changes (formatting, missing semi-colons, etc.)
- **refactor**: Code changes that neither fix a bug nor add a feature
- **test**: Adding or updating tests
- **chore**: Maintenance tasks (e.g., dependency updates)
 
Example:
```bash
feat: add new button component
```
 
 
### Known Branches:
- **develop**: Main branch for ongoing development.
- **dev1**: Deployment pipeline branch for [dev environment](https://ds-biz-tecma-dev1.tecmasolutions.com/).
- **prerelease**: Pre-production branch for staging releases.
- **release**: Production branch for live releases.
 
### Release Process:
To release changes, follow this sequence:
1. Push changes to `develop`.
2. Merge `develop` into `dev1`.
3. Merge `dev1` into `prerelease`.
4. Finally, merge `prerelease` into `release`.
 
#### Prerelease Pipeline:
To start the prerelease deployment, trigger the pipeline manually in GitLab by navigating to **CI/CD > Pipelines** and initiating it under the **stages** table.

## Running the Library Locally
 
### Installing Dependencies
To install the necessary libraries, run:
```bash
yarn
```
 
### Starting Storybook
To view and develop components locally, run:
```bash
yarn storybook
```
This will launch the Storybook server on port 6006.
 
## Available Commands
 
### **Test Commands**
> Note: Test commands are deprecated and no longer actively used.
- `yarn test`: Runs the test suite using `react-scripts`.
- `yarn test:coverage`: Generates a coverage report using `nyc`.
- `yarn test:type`: Performs type checking and linting with ESLint.
 
### **Tooling Commands**
- `yarn createComponent`: Generates a new component template.
- `yarn createStories`: Generates Storybook stories for a component.
 
### **Storybook Command**
- `yarn storybook`: Starts the Storybook server for component development.
 
### **Deprecated Commands**
- `yarn createStyle`: This command is no longer in use.
 
## Contribution Guidelines
1. Use `createComponent` and `createStories` commands to scaffold new components and stories.
2. Follow the branching strategy strictly for releases.
