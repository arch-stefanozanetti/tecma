# Design system Tecma

Catalogo **Foundations** (token visibili) e **Components** (in costruzione). Fonte di verità: **Figma**. In CSS ci sono solo **`--tecma-color-*`** e **`--tecma-typography-*`** (generati da `npm run sync:tokens`).

## Struttura sidebar

- **Introduction** — questa pagina
- **Foundations** — Colors, Typography, Radius (estendere con Spacing, Shadow…)
- **Components** — [Button](?path=/story/components-button--playground), [Icon](?path=/story/components-icon--playground), [Accordion](?path=/story/components-accordion--playground), [Alert](?path=/story/components-alert--playground), [Checkbox](?path=/story/components-checkbox--playground), [Tag](?path=/story/components-tag--playground), [Select](?path=/story/components-select--composed-single)

## Comandi

```bash
npm run build      # token → dist (opzionale; Storybook usa sorgenti TS)
npm run storybook  # http://localhost:6006
```

Se il progetto sta in un path con **`[`** o **`]`** (es. cartella `[GITHUB]`), Storybook non parte: usa un **symlink** senza parentesi verso questa cartella e lancia i comandi da lì (vedi README del package).

## Consumo in app

Package **`@tecma/design-system-tokens`**: `tokens.css`, `components.css` (inclusi Select: trigger, item, menu), export TS — vedi README.
